# 🛠 維護更新手冊 — 法庭實況還原與專業共構筆記平台

## 架構總覽

```
前端 (Next.js) → API Routes → Notion API → 6 個 Notion 資料庫
```

所有內容由 **Notion** 管理，網站自動同步。您不需要碰任何程式碼就能更新內容。

---

## 1. 修改網站文案（首頁文字/標題/公告）

**在哪改？** → Notion `Global_Settings_DB`

| 找 Key 欄位的值 | 對應位置 | 範例 |
|---|---|---|
| `Top_Banner` | 頂部綠色公告條 | ● 目前為 Beta 前導測試版 |
| `Hero_Title` | 首頁主標題 | 法庭實況還原與專業共構筆記計畫 |
| `Hero_Desc_1` | 首頁第一段文字 | 這不只是一份開庭紀錄... |
| `Hero_CTA` | 首頁按鈕文字 | 點我看現場還原！ |
| `Mission_Card_1` ~ `_3` | 計畫緣起三張卡片 | 資訊落差的消弭 |
| `Ethics_1_Title` ~ `_3_Title` | 倫理規範標題 | 嚴格去識別化 |
| `Disclaimer` | 免責聲明 | 投稿內容僅代表作者個人... |

**操作步驟：**
1. 打開 Notion → `Global_Settings_DB`
2. 找到要改的 Key（如 `Hero_Title`）
3. 修改 Content 欄位
4. 等待約 60 秒後刷新網站即可看到更新

首頁 Hero 區的輪播文字（「唯有直視真實，才能共構解方」→「讓同伴與後輩不再孤單」...）

**在哪改？** → `Global_Settings_DB` 中 Key = `Hero_TypeWriter_Texts`

- 例如：`唯有直視真實|讓同伴不再孤單|用專業視角為社工留下註腳`

> **注意：** 目前打字機文字是硬編碼在 `app/page.tsx` 中。若要改為 CMS 控制，需修改程式碼讓它從 API 取得文字。我已在程式碼中標註了位置。

## 3. 手把手教學：如何更新一場「觀庭對話紀錄」

當您要將法庭上的對話更新到平台上，您需要操作 **兩個資料庫**：先建「場次」，再塞入該場次的「對話」。

### 第一步：在 `Sessions_DB` (法庭場次) 建檔
1. 打開 Notion 進入 `Sessions_DB`。
2. 點擊右上角或最下方的 **「New (新增)」** 建立一行資料。
3. 依序填入：
   - **Session_ID (標題欄位)**：填入不會重複的代碼，例如 `S-114-1-6`。這非常重要，對話都要靠這串代碼連結！
   - **Title (文字)**：填入標題，例如 `第十次開庭：最終言詞辯論`
   - **Date (日期)**：例如 `2026-02-26`
   - **Category (單選)**：選 `刑事`
   - **Status (單選)**：這裡請先選 `草稿` (Draft)，等逐字稿都建完再改發布。
   - **Summary (文字)**：輸入一兩句話的摘要，這會顯示在前方首頁。
   - **Hot_Topic (打勾)**：若打勾，這場會在首頁被特別加上「最新進展」或閃亮標籤。

### 第二步：在 `Transcripts_DB` (法庭對話) 新增逐字句
這一步是要把法庭上每個人說的話，一句一句建檔。

1. 打開 Notion 進入 `Transcripts_DB`。
2. 點擊 **「New (新增)」** 建立對話。
   - **Line_ID (標題欄位)**：每句話的唯一編號，例如 `L-6-001` (代表第6場第1句)。
   - **Session_ID (關聯欄位)**：點擊後，從列表選擇您剛剛在第一步建立的場次（例如 `S-114-1-6`）。
   - **Role (單選)**：選發言人，例如 `法官🔴`、`檢察官🟣` 等。
   - **Action (文字，選填)**：發言時的動作，例如 `無奈地嘆氣`。
   - **Content (文字)**：對話內容，例如 `檢察官，請問你對這份報告的看法？`
   - **Order (數字)**：輸入 `1` (代表這是第 1 句)。
   - **Like_Count (數字)**：輸入 `0`。
3. 重複上述動作，把第 2 句 (Order 填 2)、第 3 句 (Order 填 3) 一路建完。

### 第三步：發布上線
1. 回到 `Sessions_DB`。
2. 找到剛剛建的那場，把 **Status** 從 `草稿` 改為 `已發布`。
3. 完成！您現在打開網站，就能看到新的觀庭對話紀錄了。

## 4. 審核留言（專業評論）

使用者在逐字稿頁面提交的留言會進入 `Interactions_DB`，預設 Status = `待審核`。

**審核流程：**
1. 打開 `Interactions_DB`
2. 篩選 Status = `待審核` 的項目
3. 閱讀 Content 欄位
4. 決定：
   - ✅ 通過 → 改 Status 為 `核准`（立即顯示在網站）
   - ❌ 拒絕 → 改 Status 為 `隱藏`（永不顯示）

---

## 5. 審核論壇投稿

使用者在論壇頁面投稿的文章進入 `Forum_DB`，預設 Status = `待審核`。

**審核流程同上：**
- `待審核` → `已發布`（顯示）
- `待審核` → `隱藏`（不顯示）

---

## 6. 查看匿名聯絡訊息

使用者透過「匿名聯絡」頁面傳送的訊息進入 `Contact_DB`。

**查看流程：**
1. 打開 `Contact_DB`
2. 新訊息的 Status = `未讀`
3. 閱讀後改為 `已讀`
4. 處理完畢改為 `已處理`

## 7. 環境變數與安全性

| 變數 | 用途 | 放在哪 |
|---|---|---|
| `NOTION_TOKEN` | Notion API 金鑰 | Vercel 後台 → Settings → Environment Variables |
| `NOTION_DB_*` | 6 個資料庫 ID | 同上 |

> ⚠️ **安全警告：**
> - `.env.local` 已被 `.gitignore` 排除，不會推到 GitHub
> - 根目錄的 `.mjs` 腳本含有硬編碼 Token，也已排除
> - **絕對不要**將 Notion Token 寫在任何會 push 到 GitHub 的檔案中

## 10. 常見問題

**Q: 改了 Notion 但網站沒更新？**
A: API 有 60 秒快取，等 1 分鐘後刷新。若仍無效，到 Vercel 後台點 Redeploy。

**Q: 被大量垃圾留言攻擊怎麼辦？**
A: 所有投稿都是「待審核」，不會自動顯示。到 Notion 全選刪除即可。API 有 IP 限流（每分鐘 5 次）和防重複按讚（IP 綁定 1 小時）。

**Q: 想加新的 CMS 欄位怎麼做？**
A: 在 `Global_Settings_DB` 新增一行，Key 填新名稱，Content 填內容。然後修改前端程式碼去讀取這個 Key。

**Q: Vercel 免費方案夠用嗎？**
A: 免費方案含 100GB 流量 + 10 萬次 API/月。對您的平台規模綽綽有餘。
