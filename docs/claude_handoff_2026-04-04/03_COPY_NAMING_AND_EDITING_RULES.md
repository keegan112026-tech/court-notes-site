# Copy, Naming, and Editing Rules

最後更新：2026-04-04

## 1. 最重要的規則

只有使用者可以定義：

- 正式文案
- 正式命名
- 敘事方式
- 概念定義

工程協作者或 AI 可以做的是：

- 版面重構
- 路由整理
- 互動優化
- 資料流調整
- UI 提升

但不可以：

- 擅自重寫使用者正式文案
- 把自己的摘要版本覆蓋使用者原文
- 把內部開發術語直接拿去當網站正式命名

## 2. 目前要特別注意的頁面

### `/about-preview`

這是目前使用者更偏好的「計畫緣起」新版本。

### `/about`

仍存在，但不一定是目前使用者偏好的正式敘事版。

### `/guide`

平台使用說明與規範頁。

### `/forum`

目前已做過文案與版型整併，需避免再把它拉回舊式「自由論壇」語言。

## 3. 不能把內部指令寫到前台

已發生過的錯誤：

- 把「人工排序」這種內部操作指令寫進前台說明卡

之後必須明確避免：

- 內部人工排序說明
- 外包驗收指令
- 資料整理流程字句
- 「這是新版／測試版」之類不必要的產品外資訊

## 4. 目前批准的 UI 原則

可做：

- 美術升級
- 微互動
- 安全增色
- 卡片化
- 結構重組

不可做：

- 為了重構而自行濃縮使用者文案
- 用泛用 AI 口吻重寫使用者敘事

## 5. 已明確排除的技術方向

除非使用者再次主動要求，否則不要再主動規劃：

- `novel`
- `yjs`

目前只保留：

- `shadcn/ui` 局部導入
- `nextjs-notion-starter-kit` 只作參考
