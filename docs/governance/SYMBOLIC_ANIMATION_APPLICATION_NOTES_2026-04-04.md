# 象徵動畫應用筆記 2026-04-04

## 目的

這份筆記用來記錄「純 CSS / SVG / 小型 React 動畫」可如何安全地套用到網站中，達到增色、提示與引導效果，同時不破壞網站本身的專業感、創傷敏感性與正文可讀性。

這份文件記錄的是：

- 原理
- 可用做法
- 可套用頁面
- 不建議使用的地方
- 後續若要實作時應遵守的安全原則

## 參考原理

這類範例的核心不是可愛風格本身，而是以下幾種技術概念：

### 1. 幾何組件拆解

把一個圖像拆成多個小部件：

- 頭
- 身體
- 尾巴
- 四肢
- 陰影

每個部件都用：

- `div`
- pseudo elements
- `border-radius`
- `transform`
- `linear-gradient`
- `box-shadow`

去構成。

這個原理可轉換成網站中的：

- 天平
- 書頁
- 筆尖
- 盾牌
- 對話泡泡
- 資料堆疊

不需要使用圖片檔，也能形成品牌化的符號。

### 2. 微幅循環動畫

範例中最值得參考的是：

- 輕微擺動
- 呼吸感
- 浮動
- 陰影縮放
- 小範圍位移

這種動畫的價值不在「秀技」，而在：

- 讓頁面有生命感
- 指示某個區塊可互動
- 強化語意而不搶戲

### 3. 互動觸發

範例使用：

- hover
- click
- state toggle

這很適合網站中需要提示但又不想寫太多說明文字的地方，例如：

- 推薦入口
- 可展開區塊
- 工作檯工具提示
- 投稿前教學入口

### 4. 象徵化而非寫實化

這類動畫最安全的做法，不是做成完整角色，而是做成「象徵物」：

- 天平代表法庭與權衡
- 筆與書頁代表書寫與共構
- 盾牌代表保護與倫理界線
- 心跳代表創傷照護
- 對話泡泡代表溝通與詮釋

這些更符合網站目前的主題氣質。

## 對目前網站的判斷

### 適合直接借鑑的部分

- 微幅呼吸感
- 卡片 hover 輕微浮起
- 象徵型 SVG / CSS 插畫
- 小型進場動畫
- 按鈕或標籤的掃光／提示感
- 低幅度狀態動效

### 不適合直接照搬的部分

- 可愛動物角色
- 大面積卡通化角色動畫
- 一直循環且視覺很強的主視覺動畫
- 對正文造成干擾的動作

原因：

這個網站處理的是法庭、社工、創傷、公共敘事與還原，不適合過度 playful 或過度裝飾。

## 可以套用在網站的地方

### A. `/sessions`

這一頁最適合導入，因為它有很多「入口型資訊卡」與「提示型資訊層」。

可用：

- 已發布完整筆記卡的微幅浮起
- 跨場次工作檯卡的柔和呼吸感
- 證人色塊的 tooltip / hover 提示
- 準備程序庭收縮區塊的箭頭動態
- 未發布場次的「整理中」狀態提示

適合加入的象徵元素：

- 書頁
- 堆疊文件
- 小型天平

### B. 首頁 Hero 右側入口卡

這裡適合低干擾但有存在感的環境動效。

可用：

- 卡片內光暈
- 推薦標籤掃光
- CTA 呼吸感
- 細微背景形狀漂浮

不適合：

- 過度誇張的 bouncing
- 角色型動畫

### C. `/about-preview` 或敘事型 prototype 頁

這類頁面適合章節前導型象徵動畫。

可用：

- 天平：問題與權衡
- 筆：書寫與建構
- 盾牌：守護與倫理
- 堆疊資料：資訊彙整

形式建議：

- 放在章節前導或標題邊
- 低速、低頻
- 不直接壓在正文上

### D. `/forum`

適合用在說明型首屏卡片。

可用：

- icon 緩慢浮動
- 入口 chips hover 提示
- 說明卡 hover 微浮起

不適合：

- 把文章列表做成太花的動畫牆

### E. `/contact`

適合用於私密傳訊與安全感提示。

可用：

- 盾牌或紙飛機微動效
- 送出按鈕掃光
- 隱私說明卡輕微浮起

## 不建議使用的地方

### 1. 正文大段落中間

正文區最重要的是可讀性，不應被動態打斷。

### 2. 管理後台主操作區

admin 頁面以清楚、穩定、快速操作為主，不應有太多花式動畫。

### 3. 工作檯主編輯區正文本身

工作檯編輯器要穩，不適合加入大量裝飾動畫。工作檯只適合：

- 狀態提示
- hover 說明
- 可點擊引用提示

## 安全原則

### 1. 動畫不能成為唯一資訊來源

即使不播放動畫，使用者仍然要能理解：

- 這是什麼
- 可不可以點
- 狀態是什麼

### 2. 動畫不能破壞可讀性

- 不蓋住文字
- 不造成閃爍
- 不讓正文位移太大

### 3. 動畫不能破壞氣質

優先使用：

- 慢
- 輕
- 穩
- 象徵性

避免：

- 可愛到稀釋議題
- 遊戲感過強
- 卡通感壓過內容

### 4. 動畫需考慮 reduced motion

後續若正式大量導入，需支援：

- `prefers-reduced-motion`

讓使用者可減少動畫干擾。

### 5. 優先用小型實作

以目前框架來說，優先順序是：

1. CSS 微動畫
2. 小型 `framer-motion`
3. 象徵型 SVG

不建議一開始就做很重的全頁動畫編排。

## 現階段建議落地順序

### 第一層：已經適合

- `/sessions` 入口卡
- `/forum` 說明卡
- `/contact` 隱私說明卡
- 首頁右側 Hero 卡

### 第二層：適合做 prototype

- `/about-preview`
- `project-intention-prototype`

### 第三層：暫緩

- 全頁大量象徵動畫
- 多角色 CSS art
- 主工作檯正文區大型動畫

## 一句話結論

這類參考最值得借用的不是角色本身，而是：

**以幾何、象徵、低幅動態去增加頁面層次與語意提示。**

對這個網站來說，最安全也最有效的做法是：

**將動畫當作語意陪襯，而不是視覺主角。**
