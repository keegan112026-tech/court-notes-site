# 新增還原筆記整合 SOP

## 用途

這份 SOP 用來處理「新的還原筆記頁面」接入網站時，哪些資料、入口與頁面需要同步更新。

此流程適用於：

- 新增單場次還原筆記
- 新增已公開工作檯頁面
- 需要人工指定排序的已發布筆記

## 核心原則

1. 還原筆記頁面新增後，不只是一份資料檔案，要同步檢查網站入口。
2. 排序不一定完全依日期，允許人工指定首頁與 `/sessions` 的展示順序。
3. 內部操作說明不得直接寫進網站正文。
4. 新增頁面後，要同時驗證「資料層、入口層、單頁層」三個層次。

## 標準流程

### 1. 新增單場次資料檔

將新的 session JSON 放入：

`data/sessions/`

命名格式：

`s-<案號或期別>.json`

最低要求：

- metadata 完整
- transcripts 可解析
- UTF-8 正常

### 2. 更新 sessions 索引

更新：

`data/sessions-index.json`

這一步決定：

- `/api/sessions` 的公開順序
- `/sessions` 輪轉櫥窗順序
- 哪一筆被視為最新

注意：

- 若有人工指定排序，應直接在 `sessions-index.json` 反映
- 不要把人工指令文案寫進前台頁面

### 3. 更新本地資料載入

更新：

`lib/local-data.ts`

需要做的事：

- import 新 session
- 加入 mapping
- 確保 `/sessions/[id]` 能正確找到資料

### 4. 驗證單頁路由

確認：

- `/sessions/<new-session-id>` 可正常開啟
- 內容可載入
- 引用、逐字稿、工作檯結構沒有壞

### 5. 更新 `/sessions` 頁面入口

檢查：

- 上方「已發布完整筆記」輪轉櫥窗是否已納入
- 排序是否正確
- 既有最新筆記是否需要維持在第一順位
- 庭期時間線中對應日期的卡片是否變成可點擊狀態

### 6. 檢查跨頁連動

視情況檢查：

- `/`
- 首頁 Hero 右側的最新公開筆記入口卡
- 首頁「最新還原場次」區
- `/sessions`
- `/sessions/[id]`
- `/api/sessions`
- 其他手動彙整頁是否需要新增對應連結

### 7. 保留人工排序與例外註記

如果這次排序不是依日期，而是人工指定：

- 記錄在交接文件或 SOP 附註
- 不要寫進前台文案
- 前台只呈現結果，不呈現內部決策過程

## 驗收清單

新增一筆還原筆記後，至少驗收以下項目：

- JSON 可解析
- 新 session 已被 `local-data.ts` 載入
- `sessions-index.json` 排序正確
- `/api/sessions` 順序正確
- `/sessions/<id>` 可開
- `/sessions` 輪轉櫥窗有出現
- `/sessions` 時間線對應日期有正確連結
- 前台沒有出現內部備註或人工排序指令

## 本次實例

以 `114/12/11` 為例，本次接入時有以下特殊處理：

- 已新增 `data/sessions/s-114-51-1211.json`
- 已加入 `data/sessions-index.json`
- 已接入 `lib/local-data.ts`
- 已同步到 `/sessions` 輪轉櫥窗
- 排序維持人工指定：
  - `2026-02-26` 第一
  - `2025-12-11` 第二

注意：上述排序規則屬於內部整合資訊，不應直接出現在網站文案中。
