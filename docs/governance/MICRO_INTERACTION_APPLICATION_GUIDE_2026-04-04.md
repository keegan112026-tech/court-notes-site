# 全站微互動增色指南 2026-04-04

## 目的

這份文件用來記錄本次「安全、低干擾、能增色」的微互動原則，避免後續在擴充動畫時失去整體語言，也避免卡通化、過度搶戲或影響可讀性。

## 核心原則

1. 動畫只能服務資訊層級，不可搶走正文主體。
2. 動畫不作為唯一資訊來源，必須在靜態狀態下也能理解。
3. 優先使用小型 motion、CSS、Radix/shadcn 的互動骨架，不做大面積重型動態。
4. 不直接搬用範例文案、角色或卡通風格，只參考其原理。
5. 深色敘事頁與工作頁的動效語氣不同：
   - 敘事頁偏氛圍、呼吸感、光暈、段落節奏
   - 工作頁偏提示、收合、懸浮、狀態回饋

## 已套用位置

### `/sessions`

- 準備程序庭改為段落級展開，不預設鋪滿所有內容。
- 證人色塊加入 tooltip 提示，讓顏色分類不只是裝飾，而是真正有資訊意義。
- 已發布筆記卡與跨場次工作檯卡保留輕微浮起與陰影變化。

### `/forum`

- 首屏兩張說明卡加入低幅度 hover 浮起與陰影提升。
- 功能入口 chips 與圓角按鈕加入輕量 hover 位移。
- 說明 icon 改為慢速上下浮動，作為視覺引子而不打斷閱讀。

### `/contact`

- 私密傳訊說明卡加入輕微浮起與盾牌呼吸感。
- 送出按鈕加入柔和掃光，作為行動提示。
- 下方三張資訊卡加入一致的 hover 浮起。

## 已評估但暫不套用

### Carousel

- 適合放在 `/sessions` 上方「已發布完整筆記」入口區。
- 建議未來形式：
  - 左：已發布場次輪播
  - 右：跨場次工作檯固定卡
- 暫時不應取代整條庭期時間線。

### SVG / CSS 象徵動態

可用原理：
- 天平、書頁、盾牌、書寫、資料堆疊、連結線

適合位置：
- 首頁右側入口卡
- `/about-preview`、`/guide-prototype` 這類敘事頁章節開頭
- 工作檯提示區與狀態區

不適合：
- 卡通角色
- 持續高頻播放的大面積動畫
- 干擾主體閱讀的裝飾

## shadcn/ui 使用紀錄

本輪已實際使用：

- `Tooltip`
  - 檔案：`components/ui/tooltip.tsx`
  - 套用在 `/sessions` 證人色塊

既有已在用：

- `Accordion`
  - 檔案：`components/ui/accordion.tsx`
  - 套用在準備程序庭的互動設計發想與收合骨架

## 明確排除

以下兩個方向除非使用者主動再次提及，否則不納入規劃：

- `novel`
- `yjs`

## 後續可延伸的安全方向

1. 首頁右側入口卡加入更穩定的微光與標籤呼吸感。
2. `/sessions` 上方入口卡未來接 Carousel。
3. `/guide-prototype`、`/about-preview` 將象徵型 SVG 動效作為章節前導，而不是正文主體。
4. 工作檯可增加引用插入與草稿狀態提示，但不更動正式文案。
