# Mechanism Policy

本文件用來固定目前網站的互動機制、輸入限制、審核邏輯與防濫用原則。

如果後續要調整，應優先修改這份文件，再修改程式。

## 1. 文章投稿

### 用途

提供使用者在單場次工作檯或跨場次工作檯撰寫觀庭共構文章，送出後進入審查流程。

### 投稿入口

- `sessions/[id]`
- `sessions/compose`

### 欄位規則

- 必填
  - `title`
  - `content`
  - `primarySessionId`
  - `sourceSessionIds`
- 可選
  - `authorName`
  - `contactEmail`

### 字數與頻率限制

- 內容字數上限：`10000` 字
- 頻率限制：同一來源 `10 分鐘最多 3 篇`

### 狀態流程

- `待審核`
- `審閱中`
- `退回修改`
- `已核准`
- `已發布`

### 其他原則

- 引用必須保留來源場次資訊
- 跨場次文章必須保留完整 `sourceSessionIds`
- `contactEmail` 不公開，只作聯繫或後續修正依據

## 2. 公開留言

### 用途

提供公開文章頁的互動回應。

### 留言入口

- `forum/[id]`

### 欄位規則

- 必填
  - `content`
  - `targetArticleId`
- 可選
  - `authorName`

### 字數與頻率限制

- 留言字數上限：`1000` 字
- 頻率限制：同一來源 `1 分鐘最多 4 則`

### 顯示與審核原則

- 留言可匿名
- 留言可填代稱
- 預設先審後上
- 留言綁定單篇公開文章，不直接作為主內容

## 3. 按讚

### 用途

作為輕量互動訊號，供文章或留言累積回應熱度。

### 基本規則

- 同一人對同一個目標只能按一次
- 不接受連續重複按讚

### 實作原則

- 基本版
  - cookie
  - local storage
  - 後端記錄已按過的 target
- 強化版
  - IP
  - 裝置指紋
  - 匿名 token

### 正式政策

- `同一瀏覽器 / 同一裝置 / 同一匿名識別` 對同一目標只能按一次

### 風險說明

在完全不登入的前提下，仍可能被：

- 清 cookie
- 換裝置
- 換 IP

繞過。

因此排行榜不得只依賴按讚數。

## 4. 私密傳訊

### 用途

處理不適合公開的資訊輸入，例如：

- 更正資訊
- 補充證據
- 敏感反映
- 私下聯繫

### 核心原則

- 此入口獨立於公開留言
- 此入口資料不公開

### 前台文案

`此為私密傳訊，您的傳訊不會公開。您可選擇是否留下姓名或信箱，若有文獻或是證據提供也歡迎。`

### 欄位規則

- 必填
  - `content`
- 可選
  - `name`
  - `email`
- 附件
  - 可上傳圖片或文件

## 5. 私密傳訊安全限制

### 字數與頻率

- 內容字數上限：`3000` 字
- 頻率限制：同一來源 `10 分鐘最多 2 次`

### 附件限制

- 單檔最大：`10MB`
- 單次最多：`3` 個檔案

### 允許格式

- 圖片
  - `jpg`
  - `jpeg`
  - `png`
  - `webp`
- 文件
  - `pdf`
  - `doc`
  - `docx`

### 不允許格式

- `exe`
- `zip`
- `js`
- `bat`
- 其他可執行檔

### 後端必做限制

- 檔名清洗
- MIME type 驗證
- 檔案大小驗證
- 上傳頻率限制

## 6. 錯誤回報與使用回饋

### 結論

網站必須提供明確的回報入口，不應只依賴一般聯絡表單。

### 建議分類

- `錯誤回報`
- `內容更正`
- `使用建議`
- `私密傳訊`
- `一般聯絡`

### 建議做法

可先共用同一個聯絡資料入口，但在分類與後台 view 上清楚區分。

後續若回報量變大，可再拆成：

- 一般聯絡 / 建議
- 私密傳訊 / 敏感反映

## 7. 排行榜與互動關聯

### 核心原則

- 主排行榜以 `公開文章` 為主
- 留言是文章的附屬互動，不應反客為主
- 按讚只是輕量訊號，不應成為唯一排序依據

### 建議指標

公開文章排行榜應綜合：

- 按讚數
- 留言數
- 最近互動時間

### 關聯邏輯

- `文章` 是主體
- `留言` 是互動層
- `按讚` 是輕量互動訊號

## 8. Notion 對應提醒

若後續以 Notion 作為主後端，至少應能承接下列資料：

- 投稿文章
- 留言
- 私密傳訊
- 錯誤回報 / 使用回饋
- 審查狀態
- 審查備註

附件上傳規則也應以前端限制為準，不直接放寬到 Notion 上限。

## 9. 目前定稿版本摘要

- 留言：`1000` 字 / `1 分鐘 4 則`
- 投稿：`10000` 字 / `10 分鐘 3 篇`
- 按讚：同目標只能按一次
- 私密傳訊：不公開、姓名與信箱可選、可附檔、需限制格式與大小
- 錯誤回報 / 使用回饋：應納入正式機制
- 排行榜：以文章為主，不只依賴按讚
