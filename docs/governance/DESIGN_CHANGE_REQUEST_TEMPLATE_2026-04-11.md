# DESIGN CHANGE REQUEST TEMPLATE - 2026-04-11

這份模板用來處理：

- 首頁板塊重排
- 子頁區塊替換
- 櫥窗 / 卡片 / CTA 位置調整
- 文案保留與替換
- 不希望 AI 把「指令」誤寫成「正式文案」的情境

目標不是增加流程負擔，而是把「頁面、位置、保留什麼、替換什麼、不能做什麼」講清楚，避免來回返工。

## 什麼情況建議一定要用

以下任一情況，建議直接用這份模板：

1. 需要調整版位，但不希望整體風格被重做
2. 需要替換區塊內容，但不希望 AI 自行改寫正式文案
3. 需要搬移既有成熟元件，而不是重新發明新板塊
4. 需求裡會提到多張截圖、附件一／附件二／附件三
5. 希望明確指定「不要做什麼」

## 命名規則

### 優先使用「位置型命名」

請優先這樣描述：

- `首頁-右櫥窗`
- `首頁-規範櫥窗`
- `筆記總覽頁-抬頭區`
- `匯集區-前導區`
- `單篇文章頁-引用預覽`

### 不建議只用附件編號

以下寫法容易造成理解偏差：

- `附件一`
- `附件二`
- `附件三`

如果必須用附件編號，請至少補一個位置說明，例如：

- `附件二（首頁-規範櫥窗上層入口）`

## 填寫模板

直接複製以下段落使用：

```md
## 設計修改卡

### 1. 頁面
- 

### 2. 目標區塊
- 

### 3. 這次要保留什麼
- 

### 4. 這次要替換成什麼
- 

### 5. 參考哪個舊版 / 哪個成熟元件
- 

### 6. 正式文案來源
- 

### 7. 明確不要做什麼
- 

### 8. 完成判準
- 
```

## 每一欄怎麼寫

### 1. 頁面

請明確指出是哪一頁：

- `首頁`
- `規範頁`
- `知識頁`
- `筆記總覽頁`
- `單場筆記頁`
- `跨場工作檯`
- `匯集區`
- `單篇文章頁`

### 2. 目標區塊

請明確指出頁面內哪一塊：

- `首頁-右櫥窗`
- `首頁-規範櫥窗`
- `首頁-主敘事下方獨立板塊`
- `筆記總覽頁-抬頭區`
- `匯集區-前導區`

### 3. 這次要保留什麼

這一欄專門避免 AI 把成熟區塊拆掉重做。

例：

- 保留原本外框
- 保留 tabs 櫥窗
- 保留卡片式板塊語法
- 保留左側標題與 CTA 的位置

### 4. 這次要替換成什麼

這一欄只描述「結構調整」，不要把正式文案混在這裡。

例：

- 把原本長文說明改成兩個長條按鈕
- 把右側 notice 區改成五個小型卡片
- 把原本並排的左右欄拆成上下兩段

### 5. 參考哪個舊版 / 哪個成熟元件

這一欄用來避免 AI 重造輪子。

例：

- 參考首頁右櫥窗那版的條狀入口
- 參考規範頁現有的三張倫理卡
- 參考首頁已完成的「資料來源與鳴謝」櫥窗

### 6. 正式文案來源

這一欄最重要。

如果正式文案要維持原樣，請直接寫：

- `文案完全沿用現有頁面`
- `以下括號內容只是指令，不是正式文案`

如果要替換成新文案，請把**真正要上站的文案**完整列在這裡。

### 7. 明確不要做什麼

這一欄專門拿來防止誤解。

例：

- 不要重做整個板塊
- 不要新增我沒提供的說明文
- 不要把我的指令句子當成正式文案
- 不要改頁首名稱
- 不要改整體色系

### 8. 完成判準

這一欄寫你看結果時最在意的事情。

例：

- 左右寬度對齊
- 板塊高度縮短
- 五個卡片可並排
- 不會出現一長一短
- 仍然看得出是同一套網站語言

## 一份寫得好的示例

```md
## 設計修改卡

### 1. 頁面
- 首頁

### 2. 目標區塊
- 首頁-規範櫥窗上層入口

### 3. 這次要保留什麼
- 保留整個成熟外框
- 保留下方內容櫥窗

### 4. 這次要替換成什麼
- 把原本較長的說明區改成兩個長條、塊狀入口

### 5. 參考哪個舊版 / 哪個成熟元件
- 參考首頁右櫥窗那版的條狀入口語法

### 6. 正式文案來源
- 先進行教學再進入觀庭筆記共構（推薦）
- 直接進入觀庭筆記共構（跨場次版面）

### 7. 明確不要做什麼
- 不要新增大段說明
- 不要重做整個大外框

### 8. 完成判準
- 區塊高度縮短
- 視覺更整齊
- 仍保有原本成熟元件風格
```

## 最後規則

1. 如果需求包含括號補充，請先明講：
   - 這是正式文案
   - 還是只是工作指令
2. 如果這次只想調版位，不想改文案，請明講：
   - `這次只動版位，不改正式文案`
3. 如果這次要沿用舊元件，不要重做，請明講：
   - `保留成熟元件，只調整位置與尺寸`
