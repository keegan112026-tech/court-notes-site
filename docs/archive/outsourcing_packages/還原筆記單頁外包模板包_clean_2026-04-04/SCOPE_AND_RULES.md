# 外包工作邊界與規則

## 核心原則

只有專案擁有者可以定義：

- 正式文案
- 命名
- 敘事
- 頁面定位

外包 AI 不得自行：

- 摘要成自己的版本
- 重寫語氣
- 補寫品牌敘事
- 替頁面命名

## 你可以做的內容

- 單一場次 metadata 整理
- 逐字稿段落切分
- 日期、程序、證人、摘要欄位整理
- 依照指定模板產出頁面所需資料
- 結構化 JSON / Markdown / 清單

## 你不能碰的內容

- 主專案路由
- Navbar / 共用頁首
- Notion 後端
- 投稿機制
- 留言機制
- 正式頁文案
- 主頁工版
- 主工程既有排序規則

## 文案規則

如果使用者沒有提供新的正式文案：

- 不要自行創作網站敘事
- 不要替內容加立場語句
- 不要把資料整理寫成宣傳文案

你應該做的是：

- 忠實整理已給內容
- 只做最小限度的中性摘要
- 不確定時明確標註

## 技術規則

- 優先產出結構化資料
- 不要假設後端 schema 以外的新欄位
- 不要新增依賴
- 不要修改現行工作檯引用機制
- 最終 JSON 必須可被標準 parser 成功解析
- 所有交付文字檔都必須為 UTF-8

## 不確定時的處理方式

若資料不足或矛盾：

- 標出不確定處
- 附上原始依據
- 不自行腦補結論
