# 新增還原筆記連動驗收流程頁

## 用途

這份文件是「新增一篇還原筆記後」的正式驗收流程頁。

目標是讓以下角色都能使用同一套標準：

- 主工程整合者
- 外包 AI
- 人工協作者

使用這份流程頁時，不需要回頭翻聊天紀錄猜測該更新哪些地方。

---

## 適用情境

當出現以下任一情境時，必須跑完這份驗收流程：

- 新增單場次還原筆記 JSON
- 新增一篇已公開的工作檯頁面
- 調整已發布筆記排序
- 新增可從 `/sessions/[id]` 直接開啟的還原筆記

---

## 驗收目標

新增一篇還原筆記後，網站應同時滿足三件事：

1. 資料可解析
2. 單頁可打開
3. 主要入口會自動跟上

如果只做到第一點或第二點，都不能算完成。

---

## 第一層：交付物驗收

### A. 原始資料檔

確認：

- 檔案已放入 `data/sessions/`
- 檔名符合命名規則
- UTF-8 正常
- JSON 可解析

必驗：

- metadata 存在
- transcripts 存在
- transcript id 唯一且連續
- 中文沒有亂碼

### B. 外包 handoff report

確認：

- 報告可正常閱讀
- 沒有亂碼
- 有清楚寫出：
  - 原始來源
  - 轉碼步驟
  - transcript 總數
  - id 範圍
  - 清理規則
  - JSON 驗證結果
  - 殘留風險

若 handoff report 不可讀，即使 JSON 可用，也應記錄缺失。

---

## 第二層：資料接線驗收

### A. sessions 索引

確認：

- `data/sessions-index.json` 已加入新筆記
- 顯示順序正確
- 若有人工指定排序，已直接反映在索引順序中

注意：

- 人工排序是內部規則
- 不得把「人工指定」「目前先這樣排」這種內部語句寫進網站正文

### B. 本地資料載入

確認：

- `lib/local-data.ts` 已 import 新資料
- `lib/local-data.ts` 已加入 mapping
- `getLocalSessionsIndex()` 與單頁資料函式都能找到新 session

### C. API

確認：

- `/api/sessions` 會回傳新 session
- 回傳順序與 `sessions-index.json` 一致

---

## 第三層：頁面與入口連動驗收

新增還原筆記後，至少要檢查以下頁面。

### 1. 單頁本身

檢查網址：

- `/sessions/<new-session-id>`

確認：

- 頁面可開啟
- 逐字內容正常
- 工作檯結構沒壞
- 引用系統仍正常

### 2. 還原筆記總覽頁

檢查網址：

- `/sessions`

確認：

- 上方「已發布完整筆記」櫥窗有納入新筆記
- 櫥窗順序正確
- 庭期時間線中，對應日期的卡片會正確變成可點擊
- 不存在額外臨時說明卡或內部備註文案

### 3. 首頁主入口

檢查網址：

- `/`

確認：

- 首頁 Hero 右側的最新公開筆記入口會跟著最新排序更新
- 首頁「最新還原場次」區會跟著最新排序更新
- 若排序有人工指定，首頁呈現的是排序結果，而不是單純依日期

### 4. 工作檯相關入口

確認：

- `/sessions/[id]` 可正常開
- `/sessions/compose` 不需因新增單篇而改動，但不得被連帶破壞

### 5. 公開文章層

如該筆後續會成為引用來源，檢查：

- `/forum`
- `/forum/[id]`

確認：

- 與工作檯的引用路徑仍有效
- 指向 session 的連結沒有失效

---

## 第四層：人工例外與決策記錄

如果這次有人工介入，例如：

- 新資料日期較早，但排序要排第二
- 首頁仍保留舊筆記為最新
- 新筆記先只出現在 `/sessions`，不進首頁

都要記錄在：

- 交接文件
- 或本文件附註

但不能直接寫成網站文案。

---

## 實際驗收清單

每次交件時，直接逐項勾選：

- [ ] `data/sessions/*.json` 已新增
- [ ] JSON 可解析
- [ ] 中文無亂碼
- [ ] handoff report 可讀
- [ ] `data/sessions-index.json` 已更新
- [ ] 排序正確
- [ ] `lib/local-data.ts` 已更新
- [ ] `/api/sessions` 含新資料
- [ ] `/sessions/<id>` 可開
- [ ] `/sessions` 上方櫥窗已更新
- [ ] `/sessions` 庭期對應卡片已變成可點擊
- [ ] 首頁 Hero 右側入口已更新
- [ ] 首頁「最新還原場次」區已更新
- [ ] 沒有把內部指令寫進前台
- [ ] 如有人工例外，已記錄在交接文件

---

## 本次範例：114/12/11

本次 `114/12/11` 接入時，應驗到：

- 新增 `data/sessions/s-114-51-1211.json`
- 已加入 `data/sessions-index.json`
- 已更新 `lib/local-data.ts`
- `/sessions/s-114-51-1211` 可開
- `/sessions` 櫥窗已出現新筆記
- 首頁主入口已能跟著 `sessions-index.json` 走
- 排序為人工指定結果：
  - `2026-02-26` 第一
  - `2025-12-11` 第二

這個人工排序只應存在於內部記錄，不應出現在前台文案。
