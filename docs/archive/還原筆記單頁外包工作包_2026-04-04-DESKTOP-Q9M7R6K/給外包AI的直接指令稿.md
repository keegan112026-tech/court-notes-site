# 給外包 AI 的直接指令稿

你現在要協助處理的是「還原筆記單頁外包工作」。

請先閱讀此資料夾內的以下檔案，再開始工作：

1. `README.md`
2. `SCOPE_AND_RULES.md`
3. `PAGE_TEMPLATE_SPEC.md`
4. `DATA_INPUT_SCHEMA.md`
5. `WORKFLOW_SOP.md`
6. `QA_CHECKLIST.md`
7. `HANDOFF_OUTPUT_TEMPLATE.md`

另外請參考這兩個檔案：

- `CURRENT_WORKSPACE_PAGE.tsx`
- `EXAMPLE_SESSION_DATA_s-114-1-6.json`

## 你的任務

你要協助整理「單一場次的還原筆記頁面資料」，讓主工程之後可以把你的成果整合回網站。

你的工作重點是：

- 整理場次 metadata
- 整理逐字稿段落
- 製作穩定的段落 id
- 把原始資料轉成結構化內容

## 你可以做的事

- 依照規格整理資料
- 將原始逐字稿切分成合理段落
- 製作 JSON 或 Markdown 中間稿
- 標出不確定處
- 依照範例格式產出可交接內容

## 你不能做的事

- 改寫正式網站文案
- 自行命名頁面
- 修改品牌敘事
- 修改主專案共用元件
- 修改路由
- 修改後端資料庫或 API
- 自行加入立場性說明文字

## 文案規則

如果使用者沒有明確提供新的正式文案：

- 不要幫網站寫新的介紹文字
- 不要把整理結果寫成宣傳文案
- 不要用你自己的語氣取代原始內容

你應該做的是：

- 忠實整理
- 中性標示
- 把不確定處留給主工程判斷

## 產出格式

請依照 `HANDOFF_OUTPUT_TEMPLATE.md` 回報。

優先產出：

- 結構化 JSON
- 或主工程可直接吸收的中間稿

不要直接覆蓋主專案檔案。

## 工作方式

1. 先理解規格與限制
2. 再看範例頁與範例資料
3. 再處理你被交辦的單一場次
4. 完成後依交付模板回報

## 品質要求

請特別檢查：

- 日期是否正確
- 段落是否遺漏
- 角色名稱是否一致
- id 是否連續且唯一
- 是否有不小心改寫原始內容

如果你遇到模糊資料，不要自行腦補結論，請直接標記不確定處。
