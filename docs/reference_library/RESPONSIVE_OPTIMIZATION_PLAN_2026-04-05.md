# 雙端 UI 優化計畫

建立日期：2026-04-05  
用途：針對桌機與手機板的後續優化方向，整理成一份可執行計畫。  
原則：先修閱讀與操作品質，再補視覺增色。

## 1. 首頁 `/`

### 目標
- 桌機：維持大氣、分區清楚、Hero 有力量
- 手機：首屏資訊更聚焦，不要一打開就壓力太大

### 優化方向
1. 桌機 Hero 的左 / 右區塊再拉順
2. 手機 Hero 收短首屏資訊密度
3. 熱門入口在手機上更像可滑讀的入口，而不是被擠壓的多卡區
4. 若之後要加輪轉櫥窗，優先考慮固定高度、卡內切換，不要讓內容整塊拉長

### 可參考
- `taxonomy`
- `magicui`
- `embla-carousel`

## 2. `/sessions`

### 目標
- 桌機：雙欄節奏穩定，資訊層級清楚
- 手機：上方案件總覽、本站須知、公開筆記入口讀起來不擁擠

### 優化方向
1. 微調上方雙欄寬度與高度比例
2. 手機上把「案件總覽 / 本站須知」再做更明確的閱讀順序
3. 時間軸與程序卡在手機上要更清楚，不要只縮小
4. 若需要櫥窗式面板，優先放在這一頁而非首頁 Hero

### 可參考
- `embla-carousel`
- `react-chrono`
- `react-vertical-timeline`

## 3. `/sessions/[id]` 工作檯

### 目標
- 桌機：維持工作效率
- 手機：不被雙欄邏輯拖垮，能清楚切換閱讀與整理

### 優化方向
1. 手機上讓 `逐字紀錄 / 共構編輯器` 切換更清楚
2. 如果需要第二層操作，優先考慮 bottom sheet / drawer，而不是再堆更多區塊
3. 強化引用插入、預覽、跳回原文的狀態感
4. 送審成功 / 失敗 / 驗證提醒應更即時

### 可參考
- `vaul`
- `tiptap`
- `react-resizable-panels`
- `sonner`

## 4. `/forum` 與文章頁

### 目標
- 桌機：更像專業文章層
- 手機：卡片摘要、段落行長、返回動線更順

### 優化方向
1. 論壇列表卡片在手機上縮得更有秩序
2. 單篇文章頁補足閱讀導引與底部延伸入口
3. 首頁熱門文章入口與論壇主頁角色區分更清楚

### 可參考
- `taxonomy`
- `blog-template`
- `kibo`

## 5. `/guide` 與 `/knowledge`

### 目標
- 桌機：更像真正的知識結構
- 手機：章節導航不迷路

### 優化方向
1. 補強頁內目錄與章節跳轉
2. 在手機上避免大段密文直接壓下來
3. 強化跨頁關聯，例如：從規範頁順利導向還原筆記與論壇

### 可參考
- `fumadocs`
- `fumadocs-ui-template`

## 6. 公共導覽 / 公共外殼

### 目標
- 桌機與手機都能維持一致品牌感
- 手機導覽不只是能打開，而是好用

### 優化方向
1. 再微調手機版 `Sheet` 導覽的層級與觸感
2. 讓公共 header / footer / metadata 更制度化
3. 長期可評估是否做真正的 public shell

### 可參考
- `shadcn-ui/ui`
- `radix-ui/primitives`
- `vaul`

## 7. 優先順序建議

### 第一優先
1. 首頁桌機 / 手機節奏收尾
2. `/sessions` 上方雙欄與手機閱讀順序
3. `/sessions/[id]` 手機切換與操作層

### 第二優先
1. `/forum` 與文章頁的雙端閱讀品質
2. `/guide` / `/knowledge` 的章節導覽
3. 公共導覽與公共外殼再制度化

### 第三優先
1. 微動態與視覺增色
2. 更進一步的互動式櫥窗
3. 高階 dashboard / workspace 功能

## 最後結論

對這個專案來說，桌機與手機優化的核心不是「兩端做兩套 UI」，而是：

- 桌機保留工作效率與完整資訊層級
- 手機保留閱讀能力與操作明確性
- 用相同品牌語言，做不同密度的呈現

後續如果要再推進首頁或工作檯，這份計畫可以直接當 responsive 方向的基底。
