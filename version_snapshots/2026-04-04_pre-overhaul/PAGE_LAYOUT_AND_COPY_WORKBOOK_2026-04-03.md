# 頁面排版與文案工作本

更新日期：2026-04-03  
用途：這是一份給使用者直接修改、補充與批註的工作本。  
目標：讓後續網頁調整時，能清楚區分「版面排版」「正式文案」「跳轉行為」「互動綁定」「後端資料關聯」。

---

## 使用方式

你可以直接在每個頁面的區塊下補充：

- 要保留的文案
- 要刪除的文案
- 想新增的文案
- 想調整的版面順序
- 想更改的按鈕位置
- 想更改的跳轉目的地
- 想保留或移動的互動功能

建議格式：

```md
[我的修改指示]
- 這一頁保留哪些區塊
- 哪個區塊要上移
- 哪個按鈕改放哪裡
- 哪段文案以我提供的新版本替換
- 哪些互動不可刪
```

## 全站原則

### 現行正式架構
- 前端本地資料：`sessions / transcripts / 引用 hover / 引用跳轉`
- Notion 後端：`Articles_DB / Comments_DB / Inbox_DB / Moderation_Log_DB`
- 正式文案、命名、敘事：由使用者提供與定義

### 禁止事項
- 未經使用者授權，不可改寫正式文案
- 不可把「為了重構」當作改寫內容的理由
- 不可因調整 A 頁面，連帶改壞 B 頁面版型與內容

### 可調整事項
- 區塊位置
- 元件排版
- 互動流程
- 跳轉邏輯
- 表單位置
- 控制台資訊呈現方式

### 頁面角色
- 網站第一入口
- 提供計畫定位
- 導向場次、文章、說明頁

### 目前版塊
1. 頂部導覽列

3. 計畫緣起／說明區塊
4. 場次／進度／公開內容導流區

- 你的首頁文案與敘事不可擅改

## 頁面關聯圖

### 主內容流
1. 首頁 `/`
2. 場次總覽 `/sessions`
3. 單場次工作頁 `/sessions/[id]`
4. 投稿進入 `Articles_DB`
5. 審查 `/admin/review`
6. 公開文章 `/forum`
7. 單篇文章 `/forum/[id]`
8. 留言進入 `Comments_DB`

### 跨場次流
1. `/sessions/compose`
2. 多場次引用
3. 投稿
4. 審查
5. 公開文章

### 聯絡／回報流
1. `/contact`
2. 寫入 `Inbox_DB`
3. 管理端 `/admin/inbox`

## 交付方式

你可以在每頁底下直接補：

- 要保留哪些區塊
- 要移動哪些區塊
- 哪些按鈕改跳哪裡
- 哪些文案換成你的新版
- 哪些互動必須保留

我之後會依這份工作本施工，避免再次出現：
- 改 A 壞 B
- 重構順手改掉你的文案
- 版面和後端互相拖累
